Was dieses Mockup zeigt: die Struktur-Entscheidungen für den Übergang von der Sanierungs-App zur vollen korodur.de — (1) der Lösungsfinder bekommt eine Eingangsweiche Neubau/Sanierung (Option C), (2) Neubau wird gleichwertige Tool-Strecke unter dem Dach „Industrieboden = Neubau + Sanierung", (3) Status der Analyse-Quick-Wins. Nicht final: Texte, Bilder, exakte Produktlogik im Neubau-Pfad (mit Frank zu schärfen). Logo/Header/Footer sind bereits vereinheitlicht — Ausnahme Katzenstreu (nur KORODUR-Schrift, CD-Vorgabe).

Reality-Check

4 · Analyse-Quick-Wins: was schon erledigt ist

Die UI/UX-Analyse lief gegen einen älteren Live-Stand. Im aktuellen Code ist das meiste bereits gelöst — der Tag gehört der Neubau-Strecke, nicht dem Nachbau erledigter Punkte.

Bereits erledigt

  • Eine Shell: ein Logo, eine Nav, ein Footer (global)
  • Eine Matrix (Produktmatrix ist raus, nur Anwendungsmatrix)
  • Globale Suche (⌘K) auf allen Seiten
  • Referenzen rendern als SSG (kein leerer No-JS-Zustand)
  • Kontakt-CTAs intern (kein Bruch auf alte korodur.de)
  • Projektart-Filter Neubau/Sanierung bei Referenzen (heute)

Heute / offen

  • Eingangsweiche + Neubau-Funnel bauen (dieses Mockup)
  • Schrittzahl vereinheitlichen (über die Weiche gelöst)
  • Positionierung Hero auf Neubau + Sanierung
  • Neubau-Referenzen importieren (Bilder/Tags — dein Track)
  • Produktbilder ergänzen (Content — dein Track)
  • CTA-Hierarchie primär/sekundär schärfen
Für morgen (Kollegen-Test): Ziel ist ein klickbarer Zwischenstand mit der neuen Struktur — Eingangsweiche, beide Funnel-Einstiege, Neubau als sichtbare Strecke. Fehlende Inhalte (Neubau-Referenzbilder, Produktbilder, finale Texte) sind als Platzhalter ok; getestet werden soll Aufbau, Navigation und Flow.
